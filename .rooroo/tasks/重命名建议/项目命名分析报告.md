# 项目命名分析报告

## 📋 执行摘要

本报告针对项目 **doubao-seedream** 进行命名分析和重新建议。通过深入分析项目代码、架构和功能特性，提供了多个更贴切的命名方案，以准确反映项目的核心价值。

---

## 🔍 项目功能分析

### 核心功能

通过分析 [`main.ts`](../../main.ts:1)、[`README.md`](../../README.md:1) 等核心文件，本项目的主要功能如下：

1. **三合一图像生成API中转服务**
   - 统一接口：提供 `/v1/chat/completions` 标准接口
   - 智能路由：根据 API Key 格式自动分发到对应渠道
   - 流式响应：支持 SSE 流式输出

2. **支持的三渠道**（详见 [`main.ts:75-97`](../../main.ts:75)）

| 渠道 | 中文名称 | API Key 格式 | 默认模型 |
|------|---------|-------------|---------|
| VolcEngine | 火山引擎 | UUID 格式 | doubao-seedream-4-0-250828 |
| Gitee | 模力方舟 | 30-60位字母数字 | z-image-turbo |
| ModelScope | 魔塔 | 以 `ms-` 开头 | Tongyi-MAI/Z-Image-Turbo |

3. **技术特性**
   - Deno 运行时
   - Docker 容器化部署
   - 完整的日志记录系统（详见 [`logger.ts`](../../logger.ts:1)）
   - 120秒超时控制
   - OpenAI API 兼容

### 架构特点

```
客户端请求 → API Key 检测 → 智能路由 → 多渠道分发
```

## ⚠️ 现有名称问题分析

### 项目名称：doubao-seedream

### 主要问题

1. **命名局限性**
   - "豆包"（doubao）仅代表火山引擎的一个模型
   - "seedream" 是豆包的特定模型名称
   - 无法体现"三渠道"的核心特点

2. **误导性**
   - 用户可能认为只支持豆包/火山引擎
   - 无法传达"统一中转"的概念
   - 与实际支持三个渠道的事实不符

3. **扩展性差**
   - 未来如果增加第四个渠道，名称将更加不合适
   - 不利于项目的长期发展和品牌建设

## 💡 新命名建议

### 方案一：强调"多渠道整合"

#### 🌟 **ImgRouter** / **图像路由器**

**英文**: `ImgRouter`

**中文**: 图像路由器

**含义**: Image Router 的缩写

**优点**:
- ✅ 简洁明了，直接体现核心功能
- ✅ 强调"路由"这一技术特点
- ✅ 不限定具体渠道，扩展性强
- ✅ 易于记忆和发音

**缺点**:
- ❌ 略偏技术化，不够文艺

**适用场景**: 适合技术向、开发者为主的项目

---

### 方案二：强调"三合一融合"

#### 🎨 **TriImage** / **三图合一**

**英文**: `TriImage`

**中文**: 三图合一

**含义**: Tri (三) + Image (图像) 的组合

**优点**:
- ✅ 突出"三渠道"的核心特点
- ✅ 简洁有力，朗朗上口
- ✅ 便于品牌传播

**缺点**:
- ❌ 固定了"三"的数量，扩展性受限
- ❌ 未来增加渠道需要改名

**适用场景**: 目前固定三个渠道的中期项目

---

### 方案三：强调"统一接口"

#### 🔗 **UniImg** / **统一图像**

**英文**: `UniImg`

**中文**: 统一图像

**含义**: Unified Image 的缩写

**优点**:
- ✅ 强调"统一接口"的优势
- ✅ 简短易记（仅6个字符）
- ✅ 扩展性极强，不限制渠道数量
- ✅ 国际化友好

**缺点**:
- ❌ 缺少情感色彩

**适用场景**: 注重可扩展性和国际化的项目

---

### 方案四：强调"智能分发"

#### 🧠 **SmartImgHub** / **智能图像枢纽**

**英文**: `SmartImgHub`

**中文**: 智能图像枢纽

**含义**: Smart + Image + Hub

**优点**:
- ✅ 强调"智能路由"技术特性
- ✅ "Hub"体现中心化分发概念
- ✅ 听起来专业且有科技感

**缺点**:
- ❌ 名称较长（11个字符）
- ❌ 略显臃肿

**适用场景**: 企业级、专业向的项目

---

### 方案五：文艺风格

## 🎯 最终推荐

### 🥇 首选推荐：**UniImg** / **统一图像**

**理由**:
1. ✅ 最简洁（仅6个字符）
2. ✅ 扩展性最强（不限制渠道数量）
3. ✅ 国际化友好
4. ✅ 准确体现"统一接口"的核心价值
5. ✅ 易于品牌化

**示例用法**:
- 仓库名：`uniimg-proxy`
- Docker镜像：`uniimg/uniimg-proxy`
- 接口端点：`http://localhost:10001/v1/chat/completions`

### 🥈 次选推荐：**ImgRouter** / **图像路由器**

**理由**:
1. ✅ 强调"智能路由"这一核心技术
2. ✅ 技术人员一看就懂
3. ✅ 简洁且专业
4. ✅ 适合技术社区传播

**示例用法**:
- 仓库名：`img-router`
- Docker镜像：`imgrouter/img-router`
- 项目描述："智能图像生成路由服务"

### 🥉 第三推荐：**ImgBridge** / **图桥**

**理由**:
1. ✅ 有意境，易于传播
2. ✅ 中英文都很简洁
3. ✅ 寓意美好（连接与传递）
4. ✅ 适合品牌化运营

**示例用法**:
- 仓库名：`img-bridge`
- Docker镜像：`imgbridge/img-bridge`
- 品牌口号："连接三大图像生成平台"
